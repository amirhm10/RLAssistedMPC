\section*{Method Goal}

The combined notebook family is the most integrated RL-assisted method in the
repo. It runs three decision layers around one MPC solve:

\begin{itemize}
\item a DQN agent chooses the horizon recipe,
\item a TD3 agent scales the internal model matrices,
\item a second TD3 agent scales the MPC cost weights.
\end{itemize}

The plant still receives a standard MPC move, but that move is generated by an
MPC object whose horizons, model, and weights may all change online.

\section*{State, Actions, Reward}

All three agents see the same scaled state:
\[
s_t =
\begin{bmatrix}
\tilde{x}_{t} \\
\tilde{y}_{\mathrm{sp},t} \\
\tilde{u}_{t}^{\mathrm{dev}}
\end{bmatrix}.
\]

The actions are heterogeneous:

\begin{itemize}
\item DQN horizon action:
      $a_t^{H} \mapsto (\Hp^{(t)},\Hc^{(t)})$
\item TD3 model action:
      $a_t^{M} \mapsto [\alpha_t, \delta_t]$
\item TD3 weight action:
      $a_t^{W} \mapsto [m_{Q_1}, m_{Q_2}, m_{R_1}, m_{R_2}]$
\end{itemize}

All three agents are trained from the same scalar reward after the MPC move has
been applied and the plant output has been observed.

\section*{Core Algorithm Flow}

\begin{enumerate}
\item Build the common scaled RL state.
\item Select or reuse a horizon action; only refresh it every
      \texttt{decision\_interval}.
\item Select model multipliers and build fresh $(A, B)$ from immutable
      baselines.
\item Select weight multipliers and update $(Q, R)$.
\item Rebuild the MPC object if the horizon pair changed, then overwrite its
      active $(A, B, Q, R)$ values regardless.
\item Solve MPC, apply the first move, step the plant, and update the observer.
\item Push transitions into whichever agents are active and call each
      \texttt{train\_step()} after warm start.
\end{enumerate}

\section*{Variants Covered}

\begin{itemize}
\item The nominal notebook runs the combined supervisor without gradual process
      drift.
\item The disturbance notebook injects the shared $Q_i$, $Q_s$, and $hA$
      schedules while reusing the same combined architecture.
\item Both notebooks log horizon selections and continuous multiplier traces so
      the three agent families can be analyzed together.
\end{itemize}

\section*{Limitations, Caveats, And Open Questions}

\begin{itemize}
\item The observer gain refresh hook exists in the function signature through
      \texttt{observer\_update\_interval}, but the actual observer-gain update
      block is currently commented out in the combined notebook.
\item Baseline fallback behavior before warm start is part of the method and
      should not be removed casually.
\item All three learners share the same scalar reward, so credit assignment is
      implicit rather than decomposed.
\item This method depends on notebook-local orchestration and should be reviewed
      there first, not in legacy helper modules.
\end{itemize}
